# slide_smith v1.2 (draft)

## Summary

v1.2 adds an **exemplar-first** (reference deck) workflow that is **LLM-free** and deterministic:

- Input: Markdown + reference PPTX
- Output: PPTX that instantiates only layouts/placeholders observed in the reference

## New commands (v1.2)

- `slide-smith analyze --reference ref.pptx --out style.profile.json`
- `slide-smith plan --input deck.md --out slide.plan.json`
- `slide-smith compile-exemplar --plan slide.plan.json --style-profile style.profile.json --out deck.spec.json`
- `slide-smith render-exemplar --reference ref.pptx --style-profile style.profile.json --deck-spec deck.spec.json --out out.pptx [--assets-base-dir .]`
- `slide-smith validate-exemplar --reference ref.pptx --pptx out.pptx --style-profile style.profile.json [--out validate.report.json]`

## Migration notes

### If you are template-first today
Continue using:

- `create` (deck spec + template package)
- `bootstrap-template`, `map-template`, `validate-template`

### If you have an exemplar deck
Use the exemplar-first pipeline:

1) `analyze` the reference deck
2) `plan` from Markdown
3) `compile-exemplar`
4) `render-exemplar`
5) `validate-exemplar`

## Known limitations (v1.2)

- Theme extraction is best-effort (fonts/colors not strictly enforced yet).
- Placeholder-only enforcement is best-effort; validator is conservative and may flag complex decks.
- Rendering images requires assets to exist on disk; use `--assets-base-dir` appropriately.
