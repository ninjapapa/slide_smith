# Design Docs

Files under `docs/design/` are historical design notes, planning docs, schemas, and internal reference material.

They are **not** the primary user-facing docs.

## Use these as user docs instead
- `docs/layout-ids.md` — supported `layout_id` values, fields, and examples
- `docs/README.md` — docs index
- `README.md` — current CLI and project overview

## What belongs under `docs/design/`
- historical design rationale
- planning documents
- schema/reference artifacts
- archived notes from earlier product shapes

If a design doc conflicts with a user doc, follow the user doc.
